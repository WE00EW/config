
# 配置规则（表格样式模板）

下面为仅包含三条规则的表格样式模板：

| 序号 | 规则类型 | 匹配 / 名称 | 字段 / 示例 | 说明 |
|---:|---|---|---|---|
| 1 | Key 配置 | 要抓取的地址 | 示例：`/v1/member` |  |
| 2 | Value 配置 | 想抓取的字段 | 示例：`x-device-id`  | 可以填写多个 用 , 隔开 示例： `x-device-id,x-device-name` |
| 3 | Token 配置 | token 字段填写 | （示例：`Token`） | 只能填写以一个 |


## 复杂规则配置（适用 value 和 key）
### 从cokie 中获取
    1. `cookie-token` 
    2. 示例： cookie-token
    3. 如果token在cookie中，就这么获取
### 从 localstorage 中获取
    1. `local-XXX`
    2. 示例：local-userData.user.userInfo.userId,local-dfeivnigceer
    3. 如果想要拿到的字段是对象有上下级关系就用 . 来找到子字段
### 从 url 的请求时候的参数获取
    1. `param-XXX`
    2. 示例：param-deviceId
    3. 如果有的字段只在请求的时候发送了，要想获取就用这种方式
